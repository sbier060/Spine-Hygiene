# Spine-IQ Brand & Style Guide

Spine-IQ is a quiet coach that lives in the menu bar. The brand should feel
**calm, precise, and alive** — a wellness instrument, not a surveillance tool
and not a medical device. Everything the user sees should reinforce: *it
watches gently, it speaks rarely, and when it speaks it's worth hearing.*

## 1. Identity

- **Name:** Spine-IQ (wordmark renders as **SpineIQ**, "IQ" in vital green).
- **Mark:** five vertebra dots tracing a lateral S-curve, head dot leading.
  Rendered from `src/components/Logo.tsx`; app icon generated by
  `scripts/gen-icons.mjs` (`npm run setup:icons`, then `npm run tauri icon
  ./app-icon.png` on macOS).
- **Voice:** behavioral, first-person-adjacent, never clinical or alarmist.
  Say "Sitting well", "Moving", "Out of frame" — never "spinal alignment",
  "diagnosis", "risk".

## 2. Color

Dark-first. The UI lives in a deep charcoal-navy; green is *earned* — it means
confirmed good posture. Red is reserved for the slouch alert. Nothing else
competes with those two meanings.

| Token | Hex | Use |
|---|---|---|
| `--bg` | `#0B0F14` | App background |
| `--surface` | `#131A22` | Cards, panels |
| `--surface-2` | `#1A2330` | Raised elements, hover |
| `--border` | `#243040` | Hairlines, dividers |
| `--text` | `#EAF1F7` | Primary text |
| `--muted` | `#8C9AAB` | Secondary text, labels |
| `--good` | `#3ECF8E` | Good posture, success (gradient to `#1F9D5F`) |
| `--good-deep` | `#1F9D5F` | Gradient base, chart fills |
| `--accent` | `#4FD1C5` | Interactive: buttons, links, focus |
| `--poor` | `#FF6B6B` | Slouch alert only |
| `--paused` | `#8C9AAB` | Paused / away / out of frame |

Rules: never use green decoratively. Alert red appears only while slouching is
confirmed. States are always color **plus** a label (accessibility).

## 3. Typography

System stack (`-apple-system, BlinkMacSystemFont, "SF Pro Text", "Helvetica
Neue", sans-serif`) — native on macOS, zero network fetches (privacy promise:
the app makes no requests after install).

| Role | Size / weight |
|---|---|
| Display (hero %) | 44px / 700, tabular numerals |
| H1 | 20px / 650 |
| Section title | 13px / 600, uppercase, letter-spacing 0.08em, muted |
| Body | 14px / 400, line-height 1.55 |
| Caption / hint | 12px / 400, muted |
| Data | tabular numerals always (`font-variant-numeric: tabular-nums`) |

## 4. Space, shape, depth

- Spacing scale: 4 / 8 / 12 / 16 / 24 / 32.
- Radius: 10px controls, 14px cards, 999px chips.
- Depth comes from **surface steps + 1px borders**, not drop shadows. One soft
  shadow tier (`--shadow-pop`) for the alert overlay only.

## 5. Motion

Calm and biological. Micro-interactions 150–250ms ease-out; the live status
dot "breathes" (4s scale/opacity loop) while monitoring is healthy. The slouch
overlay enters in 220ms and may pulse gently — it is the only urgent motion in
the product. All looping motion is disabled under `prefers-reduced-motion`.

## 6. Charts

- Percent-good-posture is the one number the brand celebrates; show it as a
  ring (today) and day bars (trend).
- Bars use the vital-green gradient at full consistency, desaturating toward
  gray-green as consistency drops; days with no data render as empty slots,
  never as zeros.
- Gridlines are hairline `--border`; values are labeled directly; numbers are
  tabular.

## 7. Don't

- No emoji as icons; inline SVG only.
- No amber "warning" states — movement is neutral, not scolded.
- No network assets (fonts, images, CDNs) — ever.
- No medical language anywhere in UI, notifications, or speech.
